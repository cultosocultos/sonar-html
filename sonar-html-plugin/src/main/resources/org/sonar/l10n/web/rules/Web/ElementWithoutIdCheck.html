This rule verifies if the id was set to the specified HTML elements.

Motivation: The id helps HTML components' identification for automated tests (Functionals and Performance).
In JSF, this id is generated at runtime when it was not set, and may be different when the code is updated.

Some examples when the id should be informed:

Noncompliant code samples

  	<input type="password" /> <!-- Id not given -->
	<input type="text" /> <!-- Id not given -->
	<input type="radio"/> <!-- Id not given -->
	<select> <!-- Id not given -->
		<option value="1">1</option>
		<option value="2">1</option>
	</select>
  	<h:form> <!-- Id not given -->
		<h:commandButton action="#{fooBean.bar}" <!-- Id not given -->
					value="#{labels['button.next']}"
					styleClass="mainButton" />
	</h:form>

Compliant Solutions

 	<input type="password" id="passwd" />
	<input type="text" id="name" /> 
	<input type="radio" id="radio:1"/> 
	<select id="choose" >
		<option value="1">1</option>
		<option value="2">1</option>
	</select>
	<h:form id="form1" > 
		<h:commandButton id="btnNext" action="#{fooBean.bar}" 
					value="#{labels['button.next']}"
					styleClass="mainButton" />
	</h:form>

Default elements that need id

 	form
 	data
 	button
 	link
 	calendar
 	select
 	input
 	inputText
 	inputTextArea
 	table
 	commandButton
 	commandLink
